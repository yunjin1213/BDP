# Spark와 Hive를 활용한 시험 기간 전후 서울 주요 야간상권 유동인구 변화 분석

**부제: 생활인구·지하철 승하차 데이터를 활용한 대학가·유흥상권 핫플 변화 분석**

## 1. 프로젝트 개요

본 프로젝트는 "시험이 끝난 밤, 서울의 대학가와 유흥상권은 실제로 붐빌까?"라는 질문에서 출발한다.

중간고사나 기말고사가 끝난 뒤 대학가, 번화가, 주점 밀집 지역의 유동인구가 증가할 것이라는 인식은 흔하다. 하지만 실제로 어느 지역의 야간 유동인구가 가장 크게 증가하는지, 대학가 상권과 일반 유흥상권의 변화가 다르게 나타나는지는 데이터로 확인할 필요가 있다.

이를 위해 서울 생활인구 데이터와 서울교통공사 지하철 시간대별 승하차 데이터를 수집하고, HDFS에 적재한 뒤 Spark와 Hive를 활용해 시험 기간 전후 서울 주요 야간상권의 유동인구 변화를 분석한다.

## 2. 문제 정의

본 프로젝트는 단순히 "시험이 끝나면 번화가가 붐빈다"는 인식을 확인하는 데 그치지 않고, 다음 질문에 답하는 것을 목표로 한다.

1. 시험 종료 후 서울 주요 야간상권의 생활인구는 실제로 증가하는가?
2. 시험 종료 후 20대 생활인구 증가율이 가장 큰 권역은 어디인가?
3. 시험 종료 후 지하철 야간 하차량이 가장 크게 증가하는 권역은 어디인가?
4. 생활인구 기준 핫플 순위와 지하철 하차량 기준 핫플 순위는 일치하는가?
5. 대학가 상권과 일반 유흥상권은 시험 기간 전후 변화 양상이 다르게 나타나는가?

생활인구와 지하철 하차량은 특정 개인이나 대학생을 직접 식별하지 않는다. 따라서 본 프로젝트에서는 "대학생 방문"을 직접 측정하는 것이 아니라, 주요 대학가·청년층 상권의 야간 유동인구 변화를 대리 지표로 분석한다.

## 3. 분석 대상 권역

초기 분석 대상은 서울의 대표적인 대학가 및 유흥상권으로 제한한다.

| 구분 | 권역 | 주요 행정동/역 |
| --- | --- | --- |
| 대학가 상권 | 신촌/이대 | 신촌동, 신촌역, 이대역 |
| 대학가·유흥상권 | 건대입구 | 화양동, 자양4동, 건대입구역 |
| 대학가·문화상권 | 혜화/대학로 | 혜화동, 혜화역 |
| 대학가 상권 | 서울대입구/샤로수길 | 행운동, 청룡동, 서울대입구역 |
| 유흥·핫플 상권 | 홍대/상수/합정 | 서교동, 합정동, 서강동, 홍대입구역, 상수역, 합정역 |
| 유흥·업무상권 | 강남/역삼 | 역삼1동, 역삼2동, 강남역, 역삼역 |
| 유흥상권 | 이태원 | 이태원1동, 이태원2동, 이태원역, 녹사평역 |

권역, 행정동, 지하철역의 연결은 직접 제작한 `area_mapping.csv`를 사용한다.

## 4. 데이터 수집

데이터 수집은 공개 데이터셋 다운로드 방식으로 수행하며, 재실행 가능한 Bash 스크립트로 자동화한다.

| 데이터 | 출처 | 수집 방식 | 활용 목적 |
| --- | --- | --- | --- |
| 서울 생활인구 데이터 | 서울 열린데이터광장 | 월별 CSV/ZIP 다운로드 | 행정동별 시간대 체류 인구 분석 |
| 서울 지하철 시간대별 승하차 데이터 | 공공데이터포털 | CSV/ZIP 다운로드 | 권역 주변역의 시간대별 승하차량 분석 |
| 권역 매핑 데이터 | 직접 제작 CSV | 수동 구축 | 권역, 행정동, 주변역 연결 |
| 시험 기간 데이터 | 직접 제작 CSV | 수동 구축 | 시험 전/중/후 기간 라벨링 |

현재 수집 대상 기간은 다음과 같다.

- 생활인구: 2025년 1월 ~ 2025년 12월, 2026년 4월
- 지하철 승하차: 2025년 상반기/하반기 일별 시간대별 승하차 데이터
- 시험 기간: 2025년 1학기 중간/기말, 2025년 2학기 중간/기말, 2026년 1학기 중간

수집 데이터는 누적 100MB 이상이어야 한다. 본 프로젝트의 원천 데이터는 생활인구와 지하철 데이터를 합쳐 약 2GB 규모로 구성된다.

## 5. 시스템 아키텍처

```text
공개 데이터 수집
  - 서울 생활인구 월별 데이터
  - 서울 지하철 일별 시간대별 승하차 데이터
  - 권역 매핑 CSV
  - 시험 기간 CSV
        ↓
Raw Data 저장
  - CSV 또는 ZIP 형태로 로컬 저장
        ↓
HDFS 적재
  - 원천 데이터를 HDFS에 업로드
        ↓
Spark 전처리
  - 인코딩 및 컬럼명 정리
  - 날짜/시간 컬럼 변환
  - 야간 시간대 필터링
  - 20대 생활인구 계산
  - 권역/행정동/지하철역 매핑
  - 시험 전/중/후 기간 라벨링
        ↓
Hive 분석 테이블 생성
  - 권역별 생활인구 테이블
  - 권역별 지하철 하차량 테이블
  - 시험 기간 분석 테이블
        ↓
Spark SQL / HiveQL 분석
  - JOIN, GROUP BY, 평균, 변화율, 순위 분석
        ↓
결과 저장 및 시각화
  - 분석 결과 CSV
  - 권역별 변화 그래프
  - 핫플 순위 결과
```

## 6. 데이터 처리 방법

Spark DataFrame을 사용해 원천 데이터를 분석 가능한 형태로 정제한다.

주요 전처리 과정은 다음과 같다.

- 생활인구 데이터의 날짜, 시간대, 행정동 코드 정리
- 남성/여성 20~24세, 25~29세 컬럼을 합산하여 20대 생활인구 계산
- 전체 생활인구 대비 20대 비중 계산
- 지하철 데이터의 역명, 호선, 승하차구분, 시간대 컬럼 정리
- 야간 시간대 필터링
  - 예: 18~21시, 21~24시, 24시 이후
- 권역 매핑 CSV와 조인
- 시험 기간 CSV와 조인하여 `before`, `during`, `after` 라벨 생성
- 권역별, 날짜별, 시간대별 생활인구와 하차량 집계

## 7. 분석 지표

### 7.1 시험 후 생활인구 증가율

```text
Population Lift =
(시험 후 야간 평균 생활인구 - 시험 전 야간 평균 생활인구)
/ 시험 전 야간 평균 생활인구
```

시험 종료 후 특정 권역의 야간 생활인구가 시험 전 대비 얼마나 증가했는지 계산한다.

### 7.2 20대 생활인구 증가율

```text
Young Population Lift =
(시험 후 야간 평균 20대 생활인구 - 시험 전 야간 평균 20대 생활인구)
/ 시험 전 야간 평균 20대 생활인구
```

전체 생활인구뿐 아니라 20대 생활인구만 따로 계산하여 청년층 변화가 더 크게 나타나는지 확인한다.

### 7.3 지하철 야간 하차량 증가율

```text
Subway Alighting Lift =
(시험 후 야간 평균 하차량 - 시험 전 야간 평균 하차량)
/ 시험 전 야간 평균 하차량
```

시험 종료 후 해당 권역 주변역으로 유입되는 인원이 증가했는지 확인한다.

### 7.4 핫플 점수

```text
Hotplace Score =
표준화된 야간 생활인구
+ 표준화된 20대 생활인구 증가율
+ 표준화된 지하철 야간 하차량 증가율
```

절대적으로 사람이 많은 곳과 시험 후 평소보다 크게 증가한 곳을 함께 고려하여 시험 후 핫플 순위를 계산한다.

## 8. 기대 결과물

- 시험 전/중/후 권역별 야간 생활인구 변화표

- 중간고사 후와 기말고사 후의 차이 분석

## 9. 실행 방법

### 9.1 데이터 수집

```bash
bash scripts/download_data.sh
bash scripts/create_reference_data.sh
```

기본 저장 위치는 레포지토리 상위의 `DataSet` 디렉토리이다.

```text
final/
├── BDP/
│   └── scripts/download_data.sh
└── DataSet/
    ├── People/
    ├── Subway/
    ├── area_mapping.csv
    └── exam_periods.csv
```

저장 위치를 변경하려면 `DATASET_DIR` 환경 변수를 지정한다.

```bash
DATASET_DIR=/path/to/DataSet bash scripts/download_data.sh
```

이미 다운로드된 파일을 다시 받으려면 `FORCE=1`을 사용한다.

```bash
FORCE=1 bash scripts/download_data.sh
```

`create_reference_data.sh`는 분석에 필요한 권역 매핑 데이터와 시험 기간 기준 데이터를 생성한다.

```text
DataSet/
├── area_mapping.csv
└── exam_periods.csv
```

### 9.2 HDFS 적재 및 분석 실행

아래 단계는 HDP Sandbox VM에서 실행한다.

```bash
# HDFS 디렉토리 생성 및 원천 데이터 업로드
bash scripts/upload_hdfs.sh

# Spark 전처리
bash scripts/run_preprocess.sh

# Hive 테이블 생성 및 분석 쿼리 실행
hive -f hive/create_tables.hql
hive -f hive/analysis_queries.hql
```

현재 레포지토리에는 데이터 수집 스크립트가 우선 포함되어 있으며, HDFS 적재, Spark 전처리, Hive 분석 스크립트는 구현 과정에서 추가한다.

## 11. 확장 가능성

본 프로젝트는 시험 기간 전후 서울 주요 야간상권의 유동인구 변화를 분석하는 데 초점을 맞추지만, 다음과 같이 확장할 수 있다.

- 대학별 실제 학사일정 데이터를 더 정밀하게 수집하여 시험 기간 정의 개선
- 카드 매출 또는 상권 업종 데이터를 결합해 실제 소비 변화 분석
- 음식점, 주점, 클럽, 카페 등 업종별 밀집도와 유동인구 변화 비교
- 날씨, 미세먼지, 공휴일, 축제 등 외부 요인을 통제 변수로 추가
- 실시간 생활인구 또는 혼잡도 API와 결합해 현재 핫플 예측 서비스로 확장
- Streamlit 또는 웹 대시보드로 권역별 변화 시각화
- 머신러닝 기반 야간 유동인구 예측 모델 추가

## 12. 한계

- 생활인구와 지하철 하차량은 대학생만을 직접 식별하지 않는다.
- 20대 생활인구는 대학생 또는 시험 종료 인원을 직접 의미하지 않는다.
- 특정 상권의 유동인구 증가가 반드시 시험 종료 때문이라고 단정할 수는 없다.
- 시험 기간은 대학마다 다르기 때문에 대표 기간으로 정의해야 한다.
- 술집, 클럽, 음식점 방문 여부를 직접 측정하는 것이 아니라 야간 유동인구를 대리 지표로 사용한다.
- 축제, 공연, 날씨, 공휴일, 지역 행사 등의 외부 요인이 결과에 영향을 줄 수 있다.
- 권역과 행정동/지하철역 매핑 과정에서 일부 오차가 발생할 수 있다.
