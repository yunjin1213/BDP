# Real-Time Air Quality Analysis Platform

## 1. 문제 정의

최근 미세먼지와 초미세먼지 문제는 시민 건강에 큰 영향을 미치고 있다.
본 프로젝트는 서울시 실시간 대기질 데이터를 수집하고,
지역별·시간대별 대기오염 패턴을 분석하는 빅데이터 플랫폼을 구축하는 것을 목표로 한다.

사용 데이터:
- 공공데이터포털 실시간 대기오염 API
- 측정소별 PM10, PM2.5, O3, NO2 데이터

분석 목표:
- 시간대별 미세먼지 변화 분석
- 지역별 대기질 비교
- 특정 시간대의 오염 집중 패턴 탐지

---

## 2. 기술 스택

- Hadoop HDFS
  - 대용량 원본 데이터 저장

- Apache Kafka
  - 실시간 대기질 데이터 스트리밍 수집

- Apache Spark
  - 데이터 전처리 및 통계 분석

- Apache Hive
  - SQL 기반 데이터 조회 및 집계

- Python
  - 데이터 수집 API 연동

- GitHub
  - 프로젝트 관리 및 협업

---

## 3. 구현 계획

### 1) 데이터 수집
- 공공데이터포털 API를 통해 실시간 대기질 데이터를 수집한다.
- Kafka Producer를 사용하여 스트리밍 데이터를 생성한다.

### 2) 데이터 저장
- Kafka Consumer가 데이터를 수신한다.
- 수신한 데이터를 Hadoop HDFS에 저장한다.

### 3) 데이터 처리
- Spark를 이용하여 결측치 제거 및 데이터 정제를 수행한다.
- 시간대별·지역별 평균 대기질 데이터를 계산한다.

### 4) 데이터 분석
- Hive 테이블을 생성하여 SQL 기반 분석을 수행한다.
- 지역별 미세먼지 농도 차이를 분석한다.
- 시간 흐름에 따른 대기질 변화를 시각화한다.

### 5) 결과 시각화
- 대기질 변화 그래프 생성
- 지역별 평균 농도 비교
- 시간대별 오염도 패턴 분석

---

## 4. 기대 효과

- 대기질 변화 패턴을 쉽게 파악할 수 있다.
- 실시간 데이터 처리 구조를 학습할 수 있다.
- Hadoop 기반 빅데이터 분석 파이프라인 경험을 얻을 수 있다.
